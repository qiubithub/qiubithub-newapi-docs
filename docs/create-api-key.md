# 创建 API Key

API 密钥（API Key）是您的应用程序、终端或项目访问 QiubiHub 服务的"通行证"。为了保障您的账户安全，我们建议为不同的用途创建独立的密钥，并妥善保管它们。

以下是创建新密钥的详细步骤。

## 创建密钥流程

### 1. 进入密钥管理页面

登录您的账户后，请看向左侧导航栏。找到并点击 **「密钥管理」** 菜单项。

点击后，您将进入密钥列表管理页面，这里展示了您当前所有的密钥状态。

![密钥管理页面](/img/1769359637221-0591f16c-03f8-4b48-9954-b7dded5ae026.png)

### 2. 开始创建新密钥

在密钥管理页面的显著位置（右上方），点击 **「创建密钥」** 按钮，系统将弹出一个新的对话框。

![创建密钥按钮](/img/1769359756068-9675aee2-72b0-48b6-8430-4a90ab0751cc.png)

### 3. 设置一个易于识别的名称

在弹出的对话框中，您需要为这个新的密钥设置一个名称。

一个好的名字能帮您在未来快速分辨出这个密钥是干什么用的。

#### 命名建议

您可以根据实际使用场景来灵活命名，例如：

- **按项目命名：** "电商网站集成"、"内部测试项目"、"个人博客"
- **按团队成员命名：** "张三的开发密钥"、"李四的测试密钥"
- **按终端设备命名：** "公司台式机"、"我的MacBook Pro"、"服务器-01"
- **按用途命名：** "Claude Code"、"生产环境"、"开发测试"

![设置密钥名称](/img/1769359001212-72e6ff30-19ac-41c9-8045-3a9e6d73c126.png)

### 4. 立即备份密钥（至关重要！）

输入名称并点击确认后，系统会生成并显示您的新密钥。

:::danger 极其重要的安全提示
**请务必现在就复制并安全地存储这个密钥！**

- 这是您 **唯一一次** 看到完整密钥的机会
- 出于安全考虑，**我们的系统不会存储您的密钥明文**
- 一旦您关闭了这个窗口或刷新了页面，您将 **永远无法再次查看** 该密钥
- 如果您不慎丢失了密钥，您将无法找回它，只能删除旧密钥并重新创建一个新的
:::

![API Key 显示](/img/1769359519775-ec280943-a7ff-4b70-9659-be686196f363.png)

**建议的存储方式：**
- 密码管理器（如 1Password、Bitwarden）
- 环境变量文件（`.env` 文件，确保不提交到 Git）
- 安全的云存储（如 iCloud Keychain）
- **切勿**: 明文存储在代码仓库、聊天记录或不安全的位置

## 限制与说明

为了保障平台安全和资源合理利用，我们对密钥数量设定了默认限制：

### 密钥数量限制

| 账户类型 | 默认限额 |
|---------|---------|
| 免费账户 | 3 个有效密钥 |
| 标准账户 | 5 个有效密钥 |
| 企业账户 | 20 个有效密钥 |

### 提升额度

如果您的业务发展需要更多的密钥数量，请随时联系我们的客户服务团队，我们将评估您的需求并为您进行调整。

### 密钥安全最佳实践

1. **定期轮换**: 建议每 3-6 个月更换一次密钥
2. **最小权限**: 为不同用途创建独立密钥，避免一个密钥泄露影响所有服务
3. **及时删除**: 不再使用的密钥应立即删除
4. **监控用量**: 定期查看各密钥的调用情况，发现异常及时处理

## 使用 API Key

### 在客户端中配置

**Claude Code:**
```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "your-api-key-here"
  }
}
```

**OpenAI SDK:**
```python
from openai import OpenAI

client = OpenAI(
    base_url="https://api.qiubithub.com/v1",
    api_key="your-api-key-here"  # 替换为你的 API Key
)
```

**cURL:**
```bash
curl https://api.qiubithub.com/v1/chat/completions \
  -H "Authorization: Bearer your-api-key-here" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Hello"}]}'
```

## 管理密钥

### 查看密钥列表

在密钥管理页面，您可以：
- 查看所有密钥的名称和创建时间
- 查看各密钥的调用统计
- 启用/禁用密钥
- 删除密钥

### 禁用密钥

如果怀疑密钥泄露或暂时不需要使用，可以先禁用密钥而不是直接删除：
- 禁用后密钥立即失效，所有使用该密钥的请求将被拒绝
- 可以随时重新启用
- 不会丢失调用历史记录

### 删除密钥

:::warning 注意
删除密钥是不可逆的操作！
:::

删除前请确保：
- 没有其他服务正在使用该密钥
- 已经更新或移除了相关配置
- 备份了需要的历史数据

## 故障排除

### 密钥无法使用

**问题**: 提示 "Invalid API Key"

**解决**:
1. 检查密钥是否完整复制（没有遗漏字符）
2. 检查密钥是否被禁用或删除
3. 检查账户余额是否充足

### 权限不足

**问题**: 提示 "Permission denied"

**解决**:
1. 检查密钥是否有权限访问该模型
2. 检查是否超出调用频率限制
3. 联系客服检查账户状态
