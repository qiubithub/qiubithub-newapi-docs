# CLI 与 API 的差异

很多用户困惑：为什么使用 Claude Code、OpenCode 等 CLI 工具时，费用似乎比直接调用 API 更高？

## 核心差异

### API 调用

直接调用 API 时，你精确控制：
- 发送什么内容
- 接收什么内容
- 调用多少次

**计费**: 仅按实际 tokens 用量计费

### CLI 工具

CLI 工具为了提供良好体验，会：
- 自动读取项目文件作为上下文
- 多次调用 API（规划、执行、验证）
- 维护对话历史

**计费**: 包含大量隐式调用

## 具体差异示例

### 场景：让 AI 解释一个函数

**直接 API 调用:**
```
输入: 50 tokens (函数代码)
输出: 200 tokens (解释)
费用: ~¥0.002
```

**Claude Code:**
```
1. 读取项目结构
2. 读取相关文件作为上下文
3. 分析函数
4. 生成解释
5. 验证输出

总 tokens: 3000+
费用: ~¥0.05
```

## 为什么 CLI 用更多 Tokens？

### 1. 上下文理解

CLI 工具会自动：
- 读取当前目录的文件
- 分析项目结构
- 加载相关依赖信息

这让 AI 能给出更准确的回答，但消耗更多 tokens。

### 2. 多轮对话

CLI 通常是交互式的：
- 规划阶段
- 执行阶段
- 验证阶段
- 错误处理

每轮都产生 API 调用。

### 3. 智能功能

高级功能如：
- 自动补全
- 代码审查
- 重构建议
- 文档生成

都涉及额外的 API 调用。

## 如何控制成本？

### 使用 CLI 时

1. **精简上下文**
   ```bash
   # 只让 AI 看特定文件
   claude -f specific-file.js
   ```

2. **限制对话长度**
   - 定期清理对话历史
   - 开启新的会话

3. **选择合适模型**
   - 简单任务用轻量级模型
   - 复杂任务用高级模型

4. **使用指令控制**
   ```bash
   # 明确限制范围
   claude "只修改这个函数，不要看其他文件"
   ```

### 替代方案

| 需求 | 低成本方案 |
|------|-----------|
| 单次问答 | 直接 API 调用 |
| 代码生成 | Web 界面 |
| 批量处理 | 脚本 + API |
| 持续开发 | 优化 CLI 配置 |

## 成本对比表

| 操作 | 直接 API | Claude Code | OpenCode |
|------|---------|-------------|----------|
| 问一个简单问题 | ¥0.002 | ¥0.05 | ¥0.03 |
| 解释一个函数 | ¥0.005 | ¥0.08 | ¥0.05 |
| 重构一个文件 | ¥0.02 | ¥0.30 | ¥0.20 |
| 生成项目文档 | ¥0.10 | ¥1.50 | ¥1.00 |

*注：以上为估算值，实际费用取决于具体任务和项目规模*

## 建议

### 预算有限时

1. **直接调用 API** 进行开发和测试
2. **使用 Web 界面** 进行探索性工作
3. **CLI 仅用于** 复杂的多步骤任务

### 追求效率时

CLI 工具虽然成本高，但能：
- 节省大量时间
- 提供更好体验
- 处理复杂任务

对于专业开发者，时间成本往往高于 API 费用。

## 总结

- CLI 工具提供更多功能，但成本更高
- 直接 API 调用最经济，但需要自己处理上下文
- 根据任务选择合适的方式
- 关注 tokens 用量，优化使用习惯
