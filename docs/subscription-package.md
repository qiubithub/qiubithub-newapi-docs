## 计费方式

我们采用 **按量计费 (Pay-as-you-go)** 模式，用多少付多少，无最低消费限制。

### 计费单位

- **Tokens**: 大语言模型的基本计费单位
  - 输入 Tokens: 你发送给模型的文本
  - 输出 Tokens: 模型返回的文本
- **1K Tokens** = 约 750 个英文单词 或 400-500 个中文字符

## 计费 FAQ

**Q: 如何查看消费明细？**

登录控制台，进入「账单中心」-「消费记录」，可查看：
- 每笔 API 调用的详细信息
- 按模型统计的用量
- 按日/周/月的汇总报表

**Q: 余额不足会怎样？**

- 首次余额不足: 发送提醒通知
- 余额为 0: API 调用将返回 402 错误（余额不足）
- 建议开启自动充值功能

**Q: 可以申请发票吗？**

可以。充值后 30 天内可在控制台申请电子发票，企业用户可申请增值税专用发票。

**Q: 计费精度如何？**

精确到小数点后 6 位，实际扣费按实际用量计算，无最低扣费限制。

**Q: 不同模型的上下文长度限制？**

| 模型 | 上下文长度 |
|------|-----------|
| GPT-4o | 128K tokens |
| Claude 3.5 Sonnet | 200K tokens |
| Gemini 1.5 Pro | 1M tokens |
| GPT-3.5-Turbo | 16K tokens |

超出上下文限制将返回错误，请在调用前检查文本长度。

## 节省成本建议

1. **选择合适的模型**: 简单任务使用轻量级模型（如 GPT-4o-mini）
2. **控制输出长度**: 通过 `max_tokens` 限制响应长度
3. **使用缓存**: 对于重复查询，在前端实现缓存机制
4. **批量处理**: 合并多个小请求为一个批量请求
5. **监控用量**: 定期查看用量报表，发现异常及时优化
