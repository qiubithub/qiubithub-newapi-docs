# 线路选择指南

QiubiHub 提供多条 API 线路，确保稳定、快速地访问各种 AI 模型。了解如何根据你的需求选择合适的线路。

## 线路类型

### 1. 直连线路 (Direct)

**特点:**
- 直接连接模型提供商的官方 API
- 最低延迟，最高稳定性
- 支持所有官方功能

**适用场景:**
- 服务器部署在境外
- 使用代理/VPN 访问
- 对延迟要求极高的应用

### 2. 国内优化线路 (CN Optimized)

**特点:**
- 专为中国大陆用户优化
- 无需代理即可访问
- 智能路由，自动选择最优路径

**适用场景:**
- 国内服务器部署
- 无代理环境
- 企业内网环境

### 3. 高并发线路 (High Concurrency)

**特点:**
- 支持更高的 QPS (每秒查询数)
- 优先处理队列
- 适合大规模应用

### 4. 备用线路 (Fallback)

**特点:**
- 当主线路故障时自动切换
- 确保服务连续性
- 无缝切换，无需修改配置

## 如何选择线路

| 你的位置 | 推荐线路 | 说明 |
|---------|---------|------|
| 中国大陆 | 国内优化线路 | 无需代理，速度最快 |
| 香港/台湾 | 直连线路或国内优化 | 两者皆可 |
| 海外 | 直连线路 | 原生连接，延迟最低 |

## 线路配置

### HTTP Header 方式

在 API 请求中添加 Header:
```
X-Route: cn-optimized
```

**支持的值:**
- direct - 直连线路
- cn-optimized - 国内优化线路
- high-concurrency - 高并发线路
- auto - 自动选择（默认）

### 环境变量方式

```bash
export QIUBI_ROUTE=cn-optimized
```

## 线路性能对比

| 线路 | 平均延迟 | 稳定性 |
|------|---------|--------|
| 直连线路 | 200-500ms | 3星 |
| 国内优化线路 | 50-150ms | 5星 |
| 高并发线路 | 80-200ms | 4星 |

## 故障排除

### 连接超时
1. 切换到国内优化线路
2. 检查网络连接
3. 联系客服检查线路状态

### 速度慢
1. 切换到距离更近的线路
2. 检查是否选择了正确的线路
3. 考虑升级到高并发线路
